# Contributing to Understanding_tech

Thank you for your interest in contributing to Understanding_tech! This is a community-driven project to help developers understand technical terminology.

## How to Contribute

### 1. Adding New Terms
- Create a new markdown file in the `docs/` directory
- Follow the template structure (Definition, Real-life Example, Use Cases, Related Terms)
- Include code examples where applicable

### 2. Improving Existing Content
- Clarify confusing explanations
- Add more real-world examples
- Fix typos or formatting issues
- Update outdated information

### 3. Adding Examples
- Place code examples in the `examples/` folder
- Include comments explaining what the code demonstrates
- Provide examples in multiple programming languages when possible

## Submission Guidelines

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/add-new-terms`)
3. Make your changes with clear, descriptive commit messages
4. Push to your fork
5. Submit a Pull Request with a description of your changes

## Style Guide

- Use clear, beginner-friendly language
- Include practical examples
- Add relevant links to related terms
- Keep explanations concise but comprehensive
- Format code blocks with proper syntax highlighting

## Code of Conduct

Be respectful and welcoming to all contributors. We're here to learn together!
